# กฎและข้อบังคับสำหรับโครงการวิเคราะห์เพลง (Homey Family Project Rules)

กฎเหล่านี้ควบคุมพฤติกรรมของเอเยนต์เมื่อผู้ใช้ส่งไฟล์เพลงใหม่ในโครงการ Homey Family:

## 1. การดำเนินการวิเคราะห์อัตโนมัติ (Automatic Analysis)
เมื่อผู้ใช้อัปโหลดเพลงใหม่ลงในโฟลเดอร์ `Song` หรือระบุถึงเพลงใหม่ ให้ดำเนินการตามขั้นตอนต่อไปนี้ทันทีโดยไม่ต้องรอคำสั่ง:
1.  **แปลงไฟล์เสียง:** หากไฟล์เป็น `.mp3` ให้รันคำสั่ง `afconvert -f WAVE -d LEI16 [input.mp3] [output.wav]` เพื่อแปลงเป็นไฟล์ `.wav` ทันที
2.  **วิเคราะห์เนื้อร้องและโครงสร้างเพลง:** เรียกใช้ Gemini API เพื่อถอดเสียง ถอดความเนื้อเพลง หาจังหวะ (BPM) และอารมณ์เพลง
3.  **จัดทำบลูปริ้นท์บอร์ดมิวสิควิดีโอ (MV Blueprint Board):** สร้างตารางสรุปรายซีน 27 ซีนที่มีการระบุรายละเอียดประเภทกล้อง แสง พรอมต์ภาพ (Photo Prompt) และพรอมต์วิดีโอ (Motion Prompt)
4.  **แยกไฟล์ผลลัพธ์:** บันทึกผลลัพธ์แยกตามเพลงโดยไม่นำไปปะปนกับเพลงอื่น ใน 3 รูปแบบไฟล์:
    *   `[ชื่อเพลง]_mv_blueprint.md`
    *   `[ชื่อเพลง]_mv_blueprint.csv`
    *   `[ชื่อเพลง]_mv_blueprint.xlsx`

## 2. ข้อกำหนดสไตล์งานภาพและตัวละคร (Visual & Character Standards)
*   **การเลือกสไตล์งานภาพ:** ห้ามตัดสินใจเลือกสไตล์ฉากหลังหรือทิศทางงานภาพเองโดยพลการ ให้ถามความคิดเห็นและตัวเลือกรูปแบบสไตล์ภาพ (เช่น ฉากหลังสีขาวล้วน, สภาพแวดล้อมเสมือนจริง, หรือสไตล์เฉพาะตัว) กับผู้ใช้ทุกครั้งก่อนเริ่มสร้างพรอมต์ในบลูปริ้นท์
*   **ตัวละคร:** ตรวจสอบความถูกต้องและสอดคล้องของชื่อตัวละครให้ตรงกันกับเอกสาร Character Sheets เสมอเมื่อมีการสร้างพรอมต์:
    *   น้องโฮมมี่ (Homey)
    *   พ่อยีราฟ (Dad)
    *   แม่หมี (Mom)
    *   คุณยายเต่า (Granny)

## 3. การปฏิสัมพันธ์กับผู้ใช้ (User Interaction)
*   หากมีบางจุดที่ยังไม่กล้าตัดสินใจเกี่ยวกับการออกแบบ ทิศทางของเรื่อง หรือสไตล์แปลกใหม่ ให้แนะนำตัวเลือกที่เป็นไปได้ พร้อมถามความคิดเห็นของผู้ใช้เพื่อขอคำยืนยัน
